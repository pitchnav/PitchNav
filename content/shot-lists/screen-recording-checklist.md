# Screen recording checklist

Record these against the real product (staging or production with a test
account), never mocked-up UI. Use a clean device frame or full-bleed capture,
consistent resolution across a single video.

- [ ] Athlete intake — profile creation form
- [ ] Video upload — selecting and uploading the side-view clip
- [ ] Processing screen — the progress/processing state
- [ ] Phase analysis — the six-phase screenshot breakdown appearing
- [ ] Scorecard — category scores / mechanics estimates view
- [ ] Drill recommendations — the drills list for a given priority
- [ ] Progress dashboard — the rolling training calendar view
- [ ] Checkout — Stripe checkout flow (use test mode; never record live card
      details)
- [ ] Completed report — the full published report, scrolled top to bottom

## Recording notes

- Record at native resolution, at least 1080p, 60fps if the source supports
  it (smoother for UI scroll/tap motion)
- Clear notifications and personal data from the test account before
  recording — no real athlete PII in marketing footage
- Move the cursor/finger deliberately — fast, jittery scrolling is hard to
  follow once cut down to a 2-3 second beat
- Capture each screen slightly longer than you think you need; it's easier
  to trim than to re-record
